Lesson 1: Introduction and Basics

Lesson 5: Variables and Strings

Variables are Used to Store Values

  • Use assignment statement to both declare and give value to a variable.
  • Follows the structure name = expression.
  • Format should consist of text and underscore, i.e. speed_of_light = 299792458
  • The advantages of variables are they can be used directly in place of long complex numbers and functions!
    • Variables can vary = once we define a variable, we can change the value of that variable. From then on, when we use the variable, the code will use the new assigned value.
    • This is quite useful because it not only provides a way to store the value of important data, but also a way to change the value of something!

Strings are Used to Store Characters

  • Follows the structure string = sequence of characters surrounded by quotes
  • Indexing Strings allows an extraction of subsequences from the string:
    • Use [] to extract a specific position from the string
    • Select subsequences from Strings by using [expression start number: stop expression number] = this selects string that is a subsequence of continuous characters from the starting to the end position



Lesson 6: Functions

Functions Take Input to do an Action

  • Function are defined by taking inputs and passing them through an operand/argument to return an output/result.
  • Defining and then using functions makes up the majority of programming
  • Once the function is defined, can call and continuously pass arguments through!
  • A function may not have an output (i.e. can simply assign a value). In python, must use a return statement in order to have output and use print command to show results.

Advantages of Functions

  • Functions allow programmers to reuse code without having to write it multiple times (just write once and run inputs through!)
  • Procedure Composition – programmers compose little/simple procedures and then use the outputs for the next procedure. Thus build simple procedures build upon one another to get complex work done.
  • Allows programmers to test easier - by building small functions to do complex work, we can more easily test small parts of the program.



Lesson 7: If & While Statements

Make Code Behave and Output Differently Based on Decisions

  • Python provides operators for making comparisons.
  • The output of a comparison is a Boolean value of True or False.
  • Examples of comparisons include:
    • <
    • >
    • == (A single equal signs means assignment, therefore have to use == for an equality comparison).
    • !=

If Statements

  • Use if statements with comparisons to then have code do an action based on decisions.
  • If statement structure is as follows:

    If comparison test expression:

    Code that will run if test is true.

  • Use indentation to show where code running ends. Python is an indent sensitive language.
  • OR compare operator evaluates the logical OR of the two operands (i.e. True or False).
  • With OR comparisons, if the first expression evaluates to True, the value is then True and thus any second expression is not evaluated.
  • In theory, can use arithmetic comparisons functions to do write every possible computer program. But this is not the most efficient way to do it.

While Loops

  • Very similar to if statement, except the test statement only executes depending if the statement is true.
  • While loops can run the test expression multiple times, as long as the evaluations statement is true.



Lesson 8: Debugging

  • Debugging is a development strategy for systemically finding and fixing errors in code.
  • When trying to use example code as base for your code, comment out the code. Then change the parts where you want your code to behave differently.
    • Make sure example code works well, otherwise your code will inherit the example code’s bugs.
    • If get frustrated, instead of throwing away first try at code in, just comment it out! That way, can return it if a later version is not as good as first try.
    • Save versions, so you can show the different versions if you need to ask for help.


Lesson 9: List & For Loops

Structured Data & Lists

  • A String is structured data = a sequence of characters.
  • List is a sequence of data types which can be anything (more powerful then string).
    • Example of List p = [‘Y’, ‘A’, ‘B’]
  • To index a list, also use [] square brackets.
  • Can make lists inside lists, thus Nested Lists!
  • Lists are versatile and can also consist of different element types. Thus, list can have combination of strings, numbers and other lists.
  • Can use index to get the position of a list within a list; use multiple brackets. [][]

Lists, Unlike Strings, Support Mutation and Aliasing

  • Mutation allows programmer to change value of list after it is created.
    • Mutation modifies the value of the elements of the list (replace the value).
      • P[0] = ‘Y’
      • Replaces/assigns first value of p list to being value of ‘y’
    • When mutating, must be considerate of all other values it effects in code!
  • Aliasing allows referrence to the same object in two different ways
  • Lists are very powerful = they can contain other lists or any other data we want, allowing us to build very complex data structures to do lots of interesting things! Other list operations include:
    • Append = append a new element to the end of the lists. Thus can add a new list to the end of a list.
    • Plus = adds two lists together to produces a new list.
    • Len = length operator and lets us know the length, or number of elements in list
    • For loop allows you to run a piece of code and repeat n number of times as opposed to a while loop
    • Index method allows to find value in list. If index does not find value, returns an error.



Lesson 10: How To Solve Problems

  • All computational problems consist of inputs and outputs. Therefore, a problem is defined by possible inputs and relationship between those inputs to outputs.
  • A solution is a procedure that can take any possible inputs and out desired results.
  • First thing first, figure out what are the inputs! Then you need to decide how to code the inputs/how they will be represented.
  • Next, understand what needs to be outputted.
  • Finally solve problem by mapping inputs to the correct outputs and understand the relationship = work out with test examples.
  • Gradually work out how to do it with examples, write algorithm that systematizes the steps and that the ideas makes sense = “pseudo code”.
  • Finally develop incrementally and test as we go!

In Summary, The Five Steps to Follow Are:

  1. Don't Panic!
  2. What Are The Inputs?
  3. What Are The Outputs?
  4. Work through some examples by hand.
  5. Simple Mechanical Solution.
  6. Develop Incrementally and Test As Go



Lesson 11: Introduction into Networks

A network is a group of entities (can be group of people, organizations, etc.) that can communicate, even though they are not all directly connected. The internet is just a special kind of network. Networks allow for non-direct communication (must be at least three entities).

What does it take to make a network?

  • A way to encode messages = be able to convert and interpret the message we want to send.
  • Way to route = need to figure out where to send the message and thus what nodes it needs to travel along.
  • Rules to decide who gets to use the network and what information can be sent

Two ways to measure a network:

  • Latency = time it takes to get from the source to destination. Measured in a unit of time such as milliseconds
  • Bandwidth = the amount of information that can be transmitted per unit of time. (The rate you can send information over. For the internet, it is often measured in megabits per second)
  • You can measure your bandwidth with online tools. In internet, there are no formal rules for who gets priority to use resource = internet is best effort service

What is a bit?

  • Smallest unit of information that can be transferred
  • Bits is a yes/no or 1/0 decision to measure information to questions
  • Can encode any number of things in bits

Protocol:

  • Protocol is a set of rules for how to entities can talk to each other. For the internet, it is rules for how client (web browser) and server can talk to each other.
  • Web Protocol is Hypertext Transfer Protocol (that is why almost all browsers start with HTTP!)
  • HTTP can send message GET (clients sends GET message to server to get a file/object and the server sends back a response which is the contents of the requested object)



Lesson 12: Make the Internet Work for You

HTML documents structure:
  • Doctype on top
  • HTML tag
  • Head tag for metadata and to references to things like stylesheets and title of the page
  • Body tag for the contents of the document

URLs:

  • URL = Uniform Resource Locator
  • HTTP is the protocol portion
  • Host is the network address like www.udacity.com The path to the document to fetch follows the address
  • Query Parameter also known as GET parameter
  • A cache (pronounced like cash) is something that stores data so that you don't have to retrieve it later. It can be used to make data requests faster.
  • Fragment is separated from the rest of the url by #
    • generally used to reference a particular part of the page looking at
    • fragment is not sent to server(it purely exists in the browser)
  • Port= in order to make an internet connection or request across the web, you need a port or an endpoint for the communication.

The request by the browser begins with a request line that looks something like GET/foo. A text line request has three main parts:

  • Method or the request making to server. The most common is get and post
  • Path = actual document we are requesting
  • Version

Request lines are followed by a number of headers. Popular ones include HOST and User-Agent (generally the browser and version)

  • User Agents is one of the most important headers; writing good and accurate user agents is courteous thing to do on the web

HTTP Responses:

  • Response looks similar to the request. Status line is the status code and then the reason phrase. Status codes examples are:
    • 200 = ok
    • 302 = found, means documented located somewhere else
    • 404 = document not found, means errors on browser side in finding doc
    • 500 = server broke in handling your request (error on server side)
  • Status line is followed by a number of headers. Examples include date, server content-type,and content-length.

Servers:

  • Purpose of a server is to respond to HTTP requests
  • Two main types of response
    • Static: pre-written file such as an image
    • Dynamic: response is built on the fly by programs called web application



Lesson 13: Forms

A form is a way to get the user to input data into the web browser which can be then sent to a server. Use the form tag element to create: <form>.

Input Element:

  • To capture the information in the form, use the element <input> in the form.
  • The input tag can take in attributes
  • For example <input name="name"> and thus name gets the value submitted in the input box in the browser.

Attribute Types:

  • Attribute type submit = <input type = "submit">. Provides a button to submit the input form.
  • Attribute action controls where a form submit to. For example, <form action=url where we want form to be submitted to>
  • Attribute type = password. Makes the input appear as dots instead of text. However this doesn’t hide it in the browser. Thus should use password type when collecting password, but know it is not sent securely to the server

Checkbox Type:

  • <Input type = "check box" name="q"> creates a checkbox called q.
  • Used for creating a toggle on or off
  • When checked, it is on for the value. If not checked off, no parameter appears in the browser.

Radio Buttons:

  • Radio buttons is another form element.
  • Example = <input type=:radio" name="q" value="value">
  • To have only one radio button be able to be selected, give them all the same name
  • Value is what checked off radio button or name is assigned, thus in the above example the q=value appears in the url

Drop Down:

  • Drop down element = <select name="q"> followed by option elements of what user is be able to select in the dropdown. For example, <option>option one</option>
  • Can also add value to options, thus <option value ="1">One</option> and thus value is the numeric digit 1

Lesson 18: Databases

What is a Database?
  • A database is a program that stores and retrieves large amounts of structured data.
  • Database can refer to the program that is storing and retrieving the data, the machine that is running the program, or it can refer to a system of machines running the program operating together.
  • Request to the servers to get information from the database can come from users through the internet.
  • Tables: columns of database make up a type; every entry in table is a row = columns go down, rows go across.
  • Generally every row will have an ID; very important because how refer to that row/information specifically.
  • Should not query database by hand: error prone, tedious, and slow.
  • Databases exist to take big chunks of data and answer queries on them in a reasonable amount of time without having to write a lot of custom code.

Types of Databases:

  • Relational database with SQL
  • Google App Engine Database
  • Amazon Dynamo
  • No SQL databases

SQL:

  • SQL= Structured Query Language = language for expressing queries; used to put data in database and then retrieving the data.
  • SQL was invented in the 1970s
  • Select is how pull/ get the data
  • Where clause is not required (use when want to put in constraints on the data retrieved)

Joins:

  • SQL type of query that uses two tables = can run query for parameters on multiples tables
  • Sequential scan = use loop to go through list and get an answer, but can be slow when there is a lot of data.
  • Can use an index to increase the speed of queries, allows for fast look of in code (like a python dictionary) because don’t have to go through every element.
  • Indexes can be used for sorting- tree data structures already have the information sorted (unlike hash tables). As a result look ups are slower and dependent on the number of elements of tree.
  • Hash tables are not sorted and have a constant look up time.

ACID = Atomicity Consistency Isolation Durability

  • Atomicity = all parts of a transaction succeed or fail together (don't just do part of it, happens together).
  • Consistency = the database will always be consistent
  • Isolation = no transaction can interfere with another
  • Durability = once a transaction is committed, it won’t be lost

Lesson 20: API

API stands for Application Programming Interface
  • APIs allows different site servers to talk to teach other through HTTP client and other protocols.
  • Using APIs, can take data from other sites in formats like XML, manipulate it and post it on your own site. This is how a lot of the internet works.

HTML is not ideal language for computers to communicate to one another

  • Difficult to parse especially if you forget to do closing tags and is error prone
  • HTML has void tags like <br> while XML does not

XML

  • Easier to parse the HTML
  • In HTML every element needs to have a closing tag while XML does not
  • In XML can make own tags
  • XML and HTML share a common lineage and thus why look similar
  • HTML can be expressed as XML

JSON

  • Structured consistent way to send/organize data in JavaScript and thus similar to XML
  • JSON is included in python 2.6 and newer
  • May use JSON to parse JavaScript and link at dictionaries and values
  • Never use eval command in python to parse JSON because there could be code that may make a mess of your computer!
  • Need convert document into JSON in order to parse in python using the load function in the JSON module

How to be a good citizen

  • If access information via API use a good user agent that specifies who you are
  • Rate-limit yourself so don’t over hit their servers with requests i.e. Use a sleep function that pauses between request
  • Public API directory = https://www.publicapis.com/
